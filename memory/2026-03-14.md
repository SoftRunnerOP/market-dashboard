# 2026-03-14: Market Dashboard Automation

- **Goal:** Built a stable, real-time market dashboard ("Mr X Dashboard") for the user.
- **Challenges:** Browser-side API calls (CoinGecko, Alternative.me) were failing due to CORS restrictions and Rate Limits, causing "Sync Error" and "undefined" states on mobile devices.
- **Final Architecture:**
    - **Back-end:** `generate_data.py` runs locally on the Mac (via cron), fetching data from Binance and CoinGecko using an API key.
    - **Storage:** Data is saved to `data.json` and pushed to GitHub.
    - **Front-end:** `index.html` (hosted on GitHub Pages) fetches the static `data.json` file. This completely eliminates CORS/API issues and enables instant, stable loading.
- **Key Learnings:** Client-side API fetching is fragile for financial dashboards. Decoupling data retrieval (Mac cron) from data display (HTML/JS) is the industrial standard for stability.
- **User Preference:** Needs 5 decimal places for DOGE, BTC/ETH price formatting, BTC Dominance, and Fear & Greed Index.
