# Bot Archive Protocol

Цель: каждый бот работает только с опорой на свой архив и не теряет контекст.

## Карта архивов
- main: `memory/bots/main.md`
- traderbot: `memory/bots/traderbot.md`
- graphicbot: `memory/bots/graphicbot.md`

## Обязательный preflight (перед ЛЮБЫМ действием)
1. Прочитать свой файл архива.
2. Извлечь 3 блока: `done` / `in_progress` / `next`.
3. Сверить входящий запрос с архивом.
4. Только после сверки формировать ответ/действие.

## Обязательная postflight-запись (после действия)
Добавить строку в свой архив в формате:
- `YYYY-MM-DD HH:MM — [tag] факт/решение/результат`

Теги:
- `[decision]` ключевое решение
- `[done]` что завершено
- `[next]` следующий шаг
- `[risk]` риск/ограничение
- `[sync]` сверка/синхронизация архива

## Fail-safe
Если архив недоступен:
- явно сообщить: "Контекст временно недоступен, работаю в ограниченном режиме".
- не придумывать факты и не симулировать историю.
